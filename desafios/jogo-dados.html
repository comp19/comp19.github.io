# Roll the dice

# Programa finalizado em sala

# Funcionalidades:

  • Simula o "lançar" do dado.
  • Permite ao utilizador 6 tentativas de acertar o número aleatório.

Obs

Os dados por enquanto funcionam como um grande dado, com intervalo de 1 a 12 — esse já seria um ponto de melhora, se consideramos que com dois dados nunca teríamos 1 como resultado.

# Código

import java.util.Scanner;

public class RollTheDice {

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        // resultado dos dados:
        int min = 1;  // valor mínimo do dado
        int max = 12; // valor máximo do dado
        int diceResult = (int) (Math.random() * ((max - min) + 1)) + min;

        int retry_limit = 6;
        int attempts = 0;

        while (attempts < retry_limit) {
            System.out.println("Please place your bet. You have "
                    + (retry_limit - attempts) + " attempts left:");
            int usersNumber = scanner.nextInt();
            if (usersNumber == diceResult) {
                // feedback para o utilizador - VENCEU
                System.out.println("Winner!");
                break;
            } else {
                // feedback para o utilizador - PERDEU
                System.out.println("Loser!");
            }
            attempts++;
        }
        scanner.close();
        if (attempts >= retry_limit) {
            System.out.println("You wasted all your attempts!");
        }
        System.out.println("The correct number was " + diceResult);
    }
}

# Objetivo

Agora vamos estender o programa... criar novas funcionalidades.
Ou alterar funcionalidades existentes — exemplo: remover as 6 tentativas se não fizer sentido para seu jogo.

Algumas ideias (apenas sugestões):

  • Poder alterar a quantidade de dados.
  • Multiplayer.
  • Aceitar apostas (apenas para simular 😃).
  • Sistema de pontuação. Exemplo em um cenário com múltiplos dados:
    • Acertar o valor individual de um dado –> 5 pontos
    • Acertar a soma (independente dos números individuais) –> 10 pontos
    • Acertar o valor individual de todos os dados (e consequentemente a soma) –> além de conseguir os pontos por ter acertado cada dado individualmente ( número de dados) e a soma dos dados, podemos também atribuir um bônus pelo feito. (10 pontos)

De novo, essas são apenas algumas prospostas...
Qualquer ideia que tenham vontade de implementar, força!
