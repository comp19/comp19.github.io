# Problem-Solving & Computational Thinking


# 1. Entender o problema

Posso explicar o problema com minhas próprias palavras?
Quais os inputs do problema?
Quais os outputs esperados da solução?
Tenho informações suficientes para resolver?
Eu já vi esse problema antes?
Começar com exemplos concretos simples.

# 2. Elaborar um plano

Pseudocódigo: Escreva explicitamente as etapas a serem seguidas.
(Nos obriga a pensar no código que será escrito e ajuda a detectar problemas conceituais antes de se aprofundar e precisar se preocupar com detalhes de sintaxe.)
Flowchart: Outra opção (ou mais uma opção) é tentar perceber visualmente a estrutura do programa.

# 3. Executar o plano

Teste cada pequeno avanço que fizer no código. (Não deixe para avaliar se existem erros quando o programa já estiver implementado por completo.)
Se não souber o que fazer... pesquise.

# 4. Rever e refatorar

Posso verificar o resultado?
Posso derivar o resultado de maneira diferente?
Posso usar o resultado ou o método para outro problema?
Posso melhorar o desempenho da minha solução?
Posso pensar em outras maneiras de refatorar?
Como outras pessoas resolveram esse problema?
